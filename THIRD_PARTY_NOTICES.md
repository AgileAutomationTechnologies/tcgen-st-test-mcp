# Third-Party Notices

## STruC++

The offline product pack includes STruC++ as a separate executable. STruC++ is published by
Autonomy / OpenPLC Project under GPL-3.0-or-later:

https://github.com/Autonomy-Logic/STruCpp

The v0.8.8 compatibility target is the TcGen STruC++ distribution
`0.5.13-tcgen.7`, based on upstream STruC++ 0.5.13. The Windows runtime
slice is pinned to AgileAutomationTechnologies commit
`cee97ecd91924b32d5371a341ef12562cc272cdd`, including the required
Windows/compiler-launch, TwinCAT short-circuit, and qualified RS/SR named-pin
compatibility fixes.

The installed `backend/libs` directory contains the runtime library assets
produced by that exact pinned checkout:

- `iec-standard-fb.stlib`
- `iec-std-functions.stlib`
- `additional-function-blocks.stlib`
- `oscat-basic.stlib`
- `iec-types.json`
- `iec-function-block-contracts.json`

These files are required for zero-configuration compilation of IEC standard
function blocks such as TON, TP, CTU, and R_TRIG. The pack also carries the
pinned STruC++ `LICENSE`, `COPYING`, and `COPYING.RUNTIME` texts beside the
backend. The STruC++ C++ runtime and standard libraries are described by the
upstream project as GPL-3.0 with the STruC++ Runtime Library Exception; release
publication remains subject to the licensing review required below.

### OSCAT Basic

`backend/libs/oscat-basic.stlib` is generated by the pinned STruC++ build from
the upstream `oscat_basic_335.library` archive (OSCAT Basic 3.35). Its exact
STruC++ wrapper metadata and source archive are retained at
`libs/sources/oscat-basic` in the pinned STruC++ source checkout. The runtime
manifest records the hash of the generated archive shipped in this pack.
Release publication must confirm the OSCAT redistribution and corresponding
notice/source obligations for that exact upstream archive.

The pinned binary archive, its checksum, and upstream source locations are
recorded in `runtime-manifest.json`.
